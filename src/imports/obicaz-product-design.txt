RÈGLES / DEMANDES PRODUIT (À INTÉGRER)
1) Accès club :
   - Non inscrit : voit très peu (au minimum une vitrine très limitée OU contenu flouté) + CTA “Rejoindre le club”.
   - Inscrit non vérifié : accès partiel, mais pas d’affichage de numéros.
   - Vérifié : téléphone belge vérifié (OTP) requis. (itsme sera plus tard → NE PAS PROMETTRE itsme en V1).
2) Contact vendeur :
   - Le vendeur saisit son numéro mais peut choisir “afficher / ne pas afficher”.
   - Si “afficher”, le numéro n’est visible qu’aux membres vérifiés.
   - Si “ne pas afficher”, contact via messages internes asynchrones (inbox), pas de chat temps réel.
3) Qualité d’annonces :
   - Prix obligatoire + prix minimum (ex : 1€ ou 5€).
   - Interdire dans titre/description : “gratuit”, “0€”, “zéro euro”, “prix sur demande”, “faire offre”, “best offer”, “à discuter” (et équivalents NL).
   - Autoriser une option claire “prix négociable” (checkbox).
   - Éviter d’utiliser le mot “gratuit” dans les CTA / UI (pas de “0€”).
4) Annonces non supprimées :
   - Statuts : Brouillon / Active / Archivées (expirées) / Vendues.
   - Une annonce expirée est “archivée” et réactivable en 1 clic (plus tard payant).
5) Bilingue BE :
   - L’utilisateur rédige en FR ou NL.
   - Afficher automatiquement la traduction sous la langue source avec label “Traduction automatique”.
   - Possibilité “modifier traduction” (optionnel V1.1).
6) IA :
   - Prévoir le flux : photos → IA génère titre + description + catégorie + attributs → utilisateur valide.
   - Ne pas promettre “estimation parfaite” : utiliser une fourchette/prix conseillé “à confirmer”.
7) Paiement :
   - V1 : packs d’annonces via Stripe (Payment Links) + codes promo (ex : 6 mois d’essai) possibles.
   - NE PAS promettre “paiement sécurisé intégré/escrow” si ce n’est pas dans V1.
8) Anti-scam :
   - Captcha/anti-bot + rate-limits (à montrer comme “anti-abus” sans détails techniques).
   - Signalement + modération (workflow simple).

STYLE MAP / RADAR (EXIGENCE)
- Utiliser une esthétique “OpenStreetMap” (carte claire, routes fines, verts pâles, gris, labels simples).
- L’écran “Radar” est un écran clé : une carte avec des épingles/markers pour :
   A) vendeurs (annonces)
   B) demandeurs (recherches / demandes)
- Couleurs des épingles selon la “vétusté” (âge restant avant expiration) :
   - Jaune = annonce “normale” (première moitié de durée)
   - Orange = annonce dans la seconde moitié de durée
   - Rouge = annonce proche de disparition (3–5 jours restants)
- Les épingles ont un pictogramme correspondant à la catégorie (vélo, manette, cheval, pinceau, etc.).
- Prévoir une légende simple + un filtre “vendeurs / demandeurs / les deux”.
